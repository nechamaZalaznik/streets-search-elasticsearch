# 🏙️ מערכת ניהול וחיפוש רחובות – באר שבע

מערכת **Full-stack** לניהול, חיפוש וסינון רחובות העיר באר שבע.  
הפרויקט מדגים שימוש מתקדם ב-**Elasticsearch** כמנוע חיפוש וביצועים גבוהים בצד הלקוח בעזרת **React**.

---

## 📋 סקירה כללית

המערכת נבנתה כדי לספק מענה לחיפוש מהיר ודינמי בנתונים עירוניים.  
היא כוללת:
- ממשק משתמש אינטואיטיבי
- שרת API מבוסס Node.js
- סקריפט לעיבוד וטעינת נתונים (Ingestion) מקובץ CSV למסד הנתונים

---

## 🛠️ טכנולוגיות בשימוש

### Frontend
- React 18  
- TypeScript  
- SASS Modules  
- Context API  

### Backend
- Node.js  
- Express  
- TypeScript  

### Database
- Elasticsearch (מנוע חיפוש מבוסס מסמכים)

### Data Handling
- CSV-Parser  
- Axios  

---

## 🏗️ ארכיטקטורה ולוגיקה

### שכבת הנתונים (Elasticsearch)
- שימוש ב-Query DSL לביצוע חיפושים מורכבים

### ניהול מדיניות מחיקה (Soft Delete)
- רשומות אינן נמחקות פיזית
- סימון באמצעות `is_active: false`
- מאפשר שמירה על שלמות הנתונים והיסטוריה

### ניהול סטייט (Context API)
- סנכרון מלא בין:
  - שדה החיפוש
  - סוג המסנן
  - טבלת התוצאות  
- ללא "רעש" של העברת Props מיותרת

---

## 🚀 הוראות הרצה

### 1. הגדרת משתני סביבה

יש ליצור קובץ `.env` בתיקיית `backend` ולהזין את הערכים הבאים:

```env
ELASTIC_NODE=your_elastic_endpoint
ELASTIC_API_KEY=your_api_key
PORT=3001
```

---

### 2. טעינת נתונים (Ingestion)

לפני הפעלת המערכת בפעם הראשונה, יש להריץ את הסקריפט שמעבד את קובץ ה-CSV ומאכלס את Elasticsearch:

```bash
cd backend
npm run ingest
```

---

### 3. הרצת השרת והאפליקציה

יש להריץ את פקודת הפיתוח בשתי תיקיות הפרויקט.

**הפעלת ה-Backend:**
```bash
cd backend
npm run dev
```

**הפעלת ה-Frontend:**
```bash
cd frontend
npm run dev
```

---

## 🔍 אפשרויות חיפוש

המערכת מציעה שלושה מנגנוני חיפוש ב-Elasticsearch:

- **חיפוש חופשי (Free Search)**  
  מבוסס Wildcard לחיפוש חלקי מילים בשם הרחוב

- **מילים מלאות (Exact Match)**  
  חיפוש מדויק של מילים בכל שדות המסמך

- **ביטוי מלא (Phrase Search)**  
  חיפוש רצף מילים מדויק תוך שימוש בשדות `keyword`
